# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**cusi** (程序员猝死提醒) - A health reminder tool for programmers to prevent overwork and late-night coding. Uses real, verified news events about programmer sudden deaths as impactful reminders during late-night work sessions.

Core principle: "注意力在哪里，提醒就在哪里" (Reminders appear where attention is focused).

## Architecture

Three independent components sharing a common event database (`data/events.json`):

1. **Claude Code Hook** (implemented) - `claude-code-hook/` - Node.js ES module, `stop` type hook
2. **Chrome Extension** (planned) - Manifest V3 + TypeScript + Vite
3. **VS Code Extension** (planned) - TypeScript + VS Code Extension API

### Claude Code Hook Runtime Flow

```
Claude response completes → stop hook triggers → hook.js runs →
  check enabled? → check night hours (22:00-05:00)? → check paused? →
  check interval (45min)? → determine reminder level → select event → stderr output
```

- Config stored at `~/.cusi/config.json`, runtime state at `~/.cusi/state.json`
- Events loaded from `data/events.json` → `~/.cusi/events.json` fallback → builtin events fallback
- Output via `console.error` (stderr) to avoid interfering with Claude's stdout
- Three-tier progressive reminders: gentle (time warning) → warning (event summary) → strong (full details)

### Event Database Schema

Events in `data/events.json` use `data/events.schema.json` (JSON Schema draft-07). Key field names:
- `id`: pattern `EVT-YYYYMMDD-XXXX` (auto-generated by workflow)
- `date_period` (not `date`): fuzzy time like "2024-Q4"
- `age_range`: enum with Chinese format like "30-35岁"
- `industry`: enum including "互联网", "游戏", "金融科技", etc.
- `source` + `source_name` (not `source_archive` alone): URL + media name
- `tags`: enum values: "加班", "熬夜", "连续工作", "忽视身体信号", "猝死", "过劳"
- `created_at`: required, ISO datetime

Events array is sorted newest-first by `created_at`. Validated by CI on push/PR.

Privacy principle: Never expose precise date + precise city + specific company + specific age together.

### Event Management Flow

```
GitHub Issue (template: event-report.yml) → Manual review → Add 'approved' label →
  GitHub Actions auto-parses issue fields → generates event → commits to data/events.json → closes issue
```

## Development Commands

### Claude Code Hook
```bash
cd claude-code-hook
npm run install-hook      # Install hook to ~/.claude/settings.json
npm run uninstall-hook    # Remove hook from settings.json
node src/cli.js status    # Check current config and night-time status
node src/cli.js test      # Display a sample reminder
node src/cli.js pause 2   # Pause for 2 hours
node src/cli.js pause tonight  # Pause until 6:00 AM
node src/cli.js set night-start 23  # Change night start hour
node src/cli.js set interval 60     # Change reminder interval (minutes)
```

### Validate Events Data
```bash
npm install -g ajv-cli ajv-formats
ajv validate -s data/events.schema.json -d data/events.json --spec=draft7 -c ajv-formats
```

## CI/CD (GitHub Actions)

- **validate-events.yml**: On push/PR touching `data/events.json` - schema validation, duplicate ID check, content length check, sort order check
- **process-approved-event.yml**: On issue labeled `approved` - parses issue body, generates event, commits, closes issue
- **check-links.yml**: Weekly cron (Sunday 00:00 UTC) + manual trigger - checks all source URLs, creates issue for broken links
- **auto-discover-events.yml**: Daily cron (08:00 UTC) + manual trigger - searches X via Grok `x_search` API for programmer sudden death events, deduplicates via OpenRouter, creates Issues for human review. Requires secrets: `GROK_API_KEY`, `OPENROUTER_API_KEY`. Configurable env: `OPENROUTER_MODEL` (default `minimax/minimax-m2.5:free`)

## Key Constraints

- **Privacy**: No data collection/upload; stats stored locally only
- **Performance**: Extension size <1MB
- **Compatibility**: Chrome/Edge latest; VS Code 1.80+; Claude Code latest
- **Offline**: Built-in base event data for offline use
- **Legal**: All events from public media with sources; include disclaimer
- **Ethics**: Strict anonymization; respect for deceased and families
