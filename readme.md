# 过劳猝死提醒 (cu-si)

## 1. 项目概述

**项目目标**：开发一个健康警示工具，通过真实、具体的过劳猝死事件，提醒所有坐在电脑前工作的人，打破"就这一次"的自我麻木循环。事件不限职业，覆盖互联网、医疗、教育、金融等各行业。

**核心理念**：注意力在哪里，提醒就在哪里。优先在用户最专注的场景（浏览器、VS Code、Claude Code 等）介入。

**定位**：警示工具 + 健康倡导工具，采用渐进式提醒策略，平衡警示效果与用户体验。

## 2. 用户画像

- **主要用户**：18-55岁坐在电脑前长时间工作的人，包括但不限于开发者、设计师、编辑、金融分析师、教师、医生等各行业从业者
- **痛点**：经常熬夜加班，明知风险却容易麻木，循环"震惊→忘记→继续熬夜"
- **使用场景**：深夜使用电脑工作时（编码、写文档、处理表格、查资料等）

## 3. 提醒策略

### 3.1 渐进式提醒机制

采用三级提醒策略，避免过于突兀：

| 级别 | 触发条件 | 内容 |
|------|----------|------|
| 温和提醒 | 首次触发 | "您已连续工作 X 小时，建议休息" |
| 警示提醒 | 忽略温和提醒后 30 分钟 | 展示猝死事件摘要 |
| 强烈警示 | 忽略警示提醒后继续工作 | 展示完整事件详情 + 强制阅读 |

### 3.2 事件选择逻辑

优先选择：
1. 最近发生的事件（时间接近）
2. 地理位置接近的事件（同城 > 同省 > 全国）
3. 避免短期内重复展示同一事件

### 3.3 正向激励机制

- 记录用户"早睡打卡"次数
- 连续 N 天在合理时间停止工作，展示鼓励信息
- 可选：生成健康报告（本周熬夜次数 vs 上周）

## 4. 核心功能需求

### 4.1 Claude Code Hook

**触发方式**：使用 `stop` 类型 hook，在 Claude 完成响应后触发检测

**触发条件**：
- 当前处于深夜时段（默认 22:00-05:00，可自定义）
- 距上次提醒已超过设定间隔

**用户控制选项**：
- 【暂停 1 小时】- 1 小时内不再提醒
- 【暂停 2 小时】- 2 小时内不再提醒
- 【今晚不再提醒】- 当日剩余时间不提醒（次日重置）
- 【调整提醒时段】- 修改默认时间范围

**实现要点**：
- 非阻塞式提醒，不打断用户工作流
- 提醒信息简洁，详情可选择展开

### 4.2 Chrome 浏览器扩展

**触发条件**（需同时满足）：
- 深夜时段（默认 22:00-05:00，可自定义）
- 用户活跃度检测：近 10 分钟内有页面交互（点击、滚动、输入、标签切换）
- 距上次提醒 45-90 分钟（随机间隔）

**位置获取**（优先级从高到低）：
1. 浏览器 Geolocation API（需用户授权，精确度高）
2. IP 定位 API（fallback，仅获取省/市级别）
3. 用户手动设置
4. 默认全国范围

**弹窗交互**：
- 使用 `chrome.notifications` 系统通知 + 可选 popup 详情页
- 温和提醒：可立即关闭
- 警示提醒：阅读 3 秒后可关闭（首次 5 秒，后续递减至 3 秒）
- 按钮选项：
  - 【查看原文】- 打开新闻来源链接
  - 【暂停 N 小时】- 选择 1/2/3 小时
  - 【我知道了】- 关闭当前提醒

**本地存储**：
- 位置信息缓存（有效期 7 天）
- 用户设置（时间段、提醒频率）
- 提醒历史（用于避免重复 + 生成统计）

### 4.3 VS Code 扩展

**触发条件**（需同时满足）：
- 深夜时段（默认 22:00-05:00，可自定义）
- 编辑活跃度：2 小时内文件保存 ≥ 5 次
- 可选加强：检测到 git commit 频率高时提高提醒频率

**提醒方式**：
- 温和提醒：`vscode.window.showInformationMessage`
- 警示提醒：`vscode.window.showWarningMessage`
- 强烈警示：自定义 Webview（展示完整事件 + 强制阅读）

**用户控制**：同 Claude Code Hook 的暂停选项

### 4.4 事件数据库

**格式**：JSON 数组，按时间倒序排列

**字段定义**：
```json
{
  "id": "string, 唯一标识",
  "date_period": "string, 模糊化日期如 '2024年下半年' 或 '2024-Q4'",
  "age_range": "string, 年龄段如 '30-35岁'",
  "industry": "string, 行业如 '互联网' '医疗' '教育' '金融'（自由填写）",
  "province": "string, 省份",
  "city": "string, 城市（可选，大城市可保留）",
  "summary": "string, 事件摘要，100字以内",
  "details": "string, 详细描述，200字以内",
  "source": "string, 新闻来源URL",
  "source_archive": "string, 存档链接（archive.org等），防止原链接失效",
  "verified": "boolean, 是否经过人工核实",
  "verified_by": "string, 核实人（可选）",
  "verified_at": "string, 核实时间",
  "tags": ["string, 标签如 '加班' '熬夜' '连续工作'"]
}
```

**隐私保护原则**：
- 不使用真实姓名，不使用"X某"格式（避免组合识别）
- 不同时暴露：精确日期 + 精确城市 + 具体公司 + 具体年龄
- 使用范围值替代精确值（年龄段、时间段）

### 4.5 事件管理流程

```
外部收集 → GitHub Issue → 人工初审 → AI 辅助核查 → 人工复核 → 合并到 JSON
```

**详细流程**：
1. **收集**：通过 GitHub Issue 模板提交事件，要求提供新闻来源
2. **初审**：维护者检查来源是否为可信媒体
3. **AI 辅助**：使用 DeepResearch 交叉验证事件真实性
4. **人工复核**：确认匿名化处理是否充分，内容是否合规
5. **自动化**：GitHub Actions 将审核通过的 Issue 转为 JSON 并更新仓库
6. **链接维护**：定期检查 source 链接有效性，失效时使用 archive 链接

## 5. 数据同步与设置

### 5.1 跨组件设置同步（可选功能）

用户可选择启用云同步，将设置同步到：
- Chrome 扩展 ↔ VS Code 扩展（通过 GitHub Gist 或自建服务）
- Claude Code Hook 使用本地配置文件，独立管理

### 5.2 本地数据统计

各组件独立记录：
- 提醒触发次数与时间
- 用户响应行为（暂停/忽略/查看详情）
- 深夜工作时长估算

可选生成周报：
- 本周深夜工作天数
- 与上周对比
- 健康建议

## 6. 非功能需求

| 类别 | 要求 |
|------|------|
| **隐私** | 不收集、不上传任何个人信息；IP 仅用于本地定位计算，不存储；统计数据仅存本地 |
| **性能** | 扩展体积 < 1MB；启动快，不影响主程序性能 |
| **可用性** | 支持深色/浅色模式；弹窗文案震撼但不惊悚过度 |
| **可维护性** | 事件数据独立 JSON 文件，便于更新；配置与代码分离 |
| **兼容性** | Chrome 最新版 + Edge；VS Code 1.80+；Claude Code 最新版 |
| **离线支持** | 内置基础事件数据，离线时使用本地缓存 |
| **国际化** | 初期仅支持中文，预留 i18n 架构 |

| 组件 | 技术选型 |

## 8. 风险与应对

| 风险 | 应对措施 |
|------|----------|
| **法律风险** | 所有事件必须来自公开媒体；注明来源；加免责声明 |
| **伦理风险** | 严格匿名化；尊重逝者及家属；定期审查内容合规性 |
| **API 不可用** | 位置 API 准备 2-3 个备选；支持离线模式 |
| **链接失效** | 使用 archive.org 存档；定期检查链接有效性 |
| **审核风险** | 避免血腥文案；提交前参考 Chrome Web Store 政策 |
| **用户反感** | 渐进式提醒；提供充分的暂停选项；正向激励平衡 |
